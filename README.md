# WendingMachineService - это веб приложение "Автомат по продаже напитков", тестовое задание для одной из компаний.  
  
## Описание:  
Веб-приложение имитирует работу автомата по продаже напитков. Напитки представлены названием, картинкой и стоимостью.  
Автомат принимает монеты номиналом в 1, 2, 5, 10 рублей. Предоставляет возможность купить напитки, внеся сумму, равную или превышающую их стоимость.  
Приложение имеет пользовательский и административный интерфейс.  
Пользовательский интерфейс предполагает покупку напитков – внесение суммы монетами, выбор напитка, получение сдачи. Доступ в пользовательский интерфейс свободен.  
Административный интерфейс предоставляет инструменты для управления автоматом.  
  
### Возможности пользовательского интерфейса  
• Вносить сумму, щелкая на кнопки с номиналом монет (1, 2, 5, 10). Если монета заблокирована, подсвечивается соответствующая кнопка и блокируется нажатие. Показывает внесенную сумму.  
• Выбирать напиток, щелкнув на соответствующую картинку. При этом не позволяет выбирать закончившийся напиток, или напиток стоимость которого превышает внесенную сумму. После выбора напитка количество оставшихся напитков уменьшается на единицу, количество монет в автомате увеличивается, оставшаяся сумма возвращается в виде сдачи. В качестве сдачи могут выдаваться заблокированные монеты.  
  
### Возможности административного интерфейса
• Администрирует ассортимент напитков: добавление, удаление напитков, изменение их количества, стоимости и изображения  
• Управлет монетами в автомате: изменение количества монет, блокирование приема тех или иных монет  
• Доступ в административный интерфейс по секретному ключу, который передаётся как параметр в адресной строке  
  
Использовались следующие технологии:  
-Asp.Net MVC 5  
-Asp.Net WebApi  
-Swagger  
-Mapper  
-EntityFramework  
-MS Sql express  
-jquery, JS, Bootstrap  

Приложение реализовано по многоуровневой архитектуре (по слоям BLL DAL WEB-UI).  
База данных лежит в папке WendingMachine\WendingMachineAPI\DataBase\WendingMachineDB.mdf.  
БД заполнена данными. При запуске проекта путь к ней подставится автоматически.  

### !!!Важно!!!  
В фале WendingMachine\WEB\appsettings.json указываются пути API, при обращении к апи url'ы берутся из настроек.!!!  
В Административный интерфейс можно попасть по параметру account/login?key=secret (пример https://localhost:44355/account/login?key=secret)  
Для выхода используйте метод account/logout (пример https://localhost:44355/account/logout)
